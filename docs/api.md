1. Функция регистрации (register)

    Путь: /api/register

    Метод: POST

    Формат вывода: при успешной регистрации(код 200):  
    json({ "message": "Успешная регистрация" })

    При ошибке, если имя занято(код 400):  
    json({ "detail": "Пользователь уже существует" })

2. Функция входа (login)

    Путь: /api/login

    Метод: POST

    Формат вывода: при успешном входе(код 200):  
    json({"token": "имя_пользователя", "is_admin": false })
    
    *(вместо `false` может быть `true`, если зашел администратор)*

    при ошибке в логине или пароле(код 400):  
    json({ "detail": "Неверный логин или пароль" })

3. Функция получения бронирований (get_bookings)
    Путь: /api/bookings/{room}  
    (вместо {room} подставляйте конкретный номер или название комнаты в URL, например: /api/bookings/room101)

    Метод: GET

    ормат вывода:При успешном запросе (код 200):
    json([{ "id": 1, "room": "название_комнаты", "username": "имя_пользователя", "booking_date": "2026-06-10", "start_time": "14:00:00", "end_time": "15:30:00" }]

    (список (массив) со всеми бронированиями для этой комнаты)  
    *(если бронирований нет, вернется просто пустой массив `[]`)*)

4. Функция создания бронирования (create_booking)
    Путь: /api/bookings

    Метод: POST

    Формат вывода: при успешном создании: json({ "id": "уникальный_id_брони", "room": "название_комнаты", "username": "имя_пользователя", "booking_date": "2026-06-10", "start_time": "14:00:00", "end_time": "15:30:00" })

    при ошибке, если время окончания некорректно:  
    json({ "detail": "Время окончания должно быть позже начала" })

    при ошибке, если выбранное время уже занято:  
    json({ "detail": "Это время уже занято" })

5. Функция удаления бронирования (delete_booking)
    Путь:/api/bookings/{booking_id}  
    (вместо {booking_id} надо подставлять уникальный ID удаляемой брони в URL, например: /api/bookings/15)

    Метод: DELETE

    Формат вывода: При успешном удалении:  
    json({ "message": "Бронь удалена" })

    При ошибке, если бронь с таким ID не существует:  
    json({ "detail": "Бронь не найдена" })

    При ошибке, если обычный пользователь пытается удалить чужую бронь:  
    json({ "detail": "Вы не можете удалить чужую бронь" })

6. Функция получения списка комнат (get_rooms)

    Путь:/api/rooms

    Метод: GET

    Формат вывода: При успешном запросе возвращается простой массив строк с названиями всех доступных комнат:  
    json([ "Конференц-зал", "Переговорная 1", "Кабинет 302" ])  
    *(если в базе данных еще нет ни одной комнаты, вернется пустой массив `[]`)*

7. Функция получения списка пользователей (get_users)

    Путь: /api/users

    Метод: GET

    Формат вывода: При успешном запросе возвращается простой массив строк с именами (логинами) всех зарегистрированных пользователей:  
    json([ "ivan_ivanov", "alex", "maria_p" ])  
    *(если пользователей в базе нет, вернется пустой массив `[]`)*
